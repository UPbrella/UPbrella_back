## 📜 브랜치 룰

### Prefix

`feature/` : 기능 개발 작업과 관련된 브랜치를 의미합니다.

`fix/` : 버그 수정 작업과 관련된 브랜치를 의미합니다.

`refactor/` : 코드 품질 개선 작업과 관련된 브랜치를 의미합니다.

`release/` : 배포 작업과 관련된 브랜치를 의미합니다.

### Suffix

`이슈번호 #000 + 기능 명`

### 브랜치 예제

**`feature/#1-header`**
